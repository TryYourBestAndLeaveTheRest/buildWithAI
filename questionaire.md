# Feedback Form Question Ideas

Use this as a menu. You do not need to include everything in one form.

## Recommended Structure (Short Form)
1. Overall satisfaction (1-5)
2. What were you trying to do today?
3. Was it easy to complete? (Yes/No)
4. Biggest issue faced (if any)
5. One feature you want next
6. Would you use this app again? (Yes/No/Maybe)
7. Optional contact email

## Core Questions (Must-Have)

### 1) Satisfaction
- How satisfied are you with your experience today?
- Type: Linear scale (1-5 or 1-10)
- Label idea: 1 = Very dissatisfied, 5 = Very satisfied

### 2) Task Success
- Did you successfully complete what you came here to do?
- Type: Multiple choice
- Options: Yes, Partly, No

### 3) Ease of Use
- How easy was it to navigate the app?
- Type: Linear scale (1-5)

### 4) Top Pain Point
- What was the most frustrating part of your experience?
- Type: Paragraph

### 5) Feature Request
- What is one feature we should build next?
- Type: Short answer

## Product-Specific Questions (Campus Swap & Drop)

### Marketplace Experience
- What were you trying to do today?
- Type: Multiple choice
- Options: Post an item, Find an item, Negotiate/Bargain, Message/Contact, Browse only

### Listing Quality
- Were listings clear enough (title, condition, price, details)?
- Type: Multiple choice
- Options: Very clear, Somewhat clear, Not clear

### Bargaining Flow
- How smooth was the bargaining process?
- Type: Linear scale (1-5)

### Trust & Safety
- Did you feel safe using this platform?
- Type: Multiple choice
- Options: Yes, Mostly, Not sure, No

### Notification Quality
- Were notifications helpful and timely?
- Type: Multiple choice
- Options: Very helpful, Somewhat helpful, Not helpful, I did not notice notifications

## UX and Performance Questions

### Speed
- How fast did pages load for you?
- Type: Multiple choice
- Options: Very fast, Acceptable, Slow, Very slow

### Device Experience
- Which device are you using?
- Type: Multiple choice
- Options: Android, iPhone, Laptop/Desktop, Tablet

### Visual Clarity
- Was anything confusing in the interface (buttons, labels, screens)?
- Type: Paragraph

## Prioritization Questions (Great for Roadmap)

### Most Important Improvement
- Which improvement matters most to you right now?
- Type: Multiple choice (checkbox optional)
- Options:
  - Faster loading speed
  - Better search/filter
  - Better chat/communication
  - Better notifications
  - Better listing quality controls
  - Safer transactions

### Value Question
- If we improved one thing this month, what should it be?
- Type: Short answer

## Optional Segmentation Questions

### User Type
- Are you mostly a buyer, seller, or both?
- Type: Multiple choice
- Options: Buyer, Seller, Both

### Frequency
- How often do you use the app?
- Type: Multiple choice
- Options: First time, Weekly, Monthly, Rarely

### Recommendation
- How likely are you to recommend this app to a friend?
- Type: Linear scale (0-10)

## Smart Conditional Logic Ideas

### If user answers "No" to task success:
Show:
- What blocked you from completing your task?
- Where did it happen? (Page/screen name)

### If user gives low rating (1-2):
Show:
- What should we fix first to improve your experience?

### If user gives high rating (4-5):
Show:
- What did you like most?
- Would you like early access to new features?

## Two Ready-to-Use Templates

### A) Ultra-Short (60-90 seconds)
1. Satisfaction (1-5)
2. Did you complete your goal? (Yes/Partly/No)
3. Biggest issue (short answer)
4. One thing to improve (short answer)

### B) Standard (2-3 minutes)
1. Satisfaction (1-5)
2. What were you trying to do? (MCQ)
3. Ease of navigation (1-5)
4. App speed (MCQ)
5. Bargaining flow smoothness (1-5)
6. Biggest frustration (paragraph)
7. Feature request (short answer)
8. Likelihood to recommend (0-10)
9. Optional email for follow-up

## Tips
- Keep required questions between 3 and 5 only.
- Put open-text questions near the end.
- Avoid more than 1 long paragraph field.
- Use one clear question per screen/section.
- Review responses weekly and tag by theme (speed, UX, trust, features).
